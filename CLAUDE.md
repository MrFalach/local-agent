# Local Agent — הוראות ניתוב

## כלל בסיסי

לעבודת קוד בנפח (קובץ חדש, שלד, ריפקטור גדול) — קרא תחילה ל-`local-agent:dev`,
קבל את הפלט, והשתמש בו כבסיס לעריכה. לעריכות קטנות וממוקדות — ערוך ישירות; אין טעם
לעקוף דרך הכלי בשביל שינוי של כמה שורות (זה רק מוסיף השהיה).

## מתי לקרוא לאיזה כלי

### `local-agent:dev` — ייצור קוד (מקומי, חינם)
- כתיבת פונקציות, קלאסים, קומפוננטות
- שלד של קובץ חדש
- ריפקטורינג של קוד קיים
- כתיבת בדיקות יחידה
- הוספת תיעוד לקוד

### `local-agent:ask` — שאלות טכניות (מנותב אוטומטית)
- שאלות על ספריות וכלים
- השוואה בין גישות
- פירוש שגיאות פשוטות

### `local-agent:cloud` — כפה ענן (רק כשנחוץ)
- ארכיטקטורה של מערכת מורכבת
- אבטחה ו-security review
- תקלות עמוקות ו-race conditions
- החלטות עיצוב קריטיות

### `local-agent:local` — כפה מקומי
- כשאתה בטוח שהמשימה פשוטה וברצונך לחסוך

## זרימת עבודה לכתיבת קוד

1. קרא ל-`local-agent:dev` עם תיאור המשימה
2. קבל את הקוד שהמודל המקומי ייצר
3. ערוך אותו לקובץ עם `Edit` / `Write`
4. הוסף שיפורים והתאמות בעצמך אם נדרש

## חשוב

- לייצור קוד בנפח (מעל ~30 שורות) עדיף לעבור דרך `local-agent:dev`; עריכות קטנות — ישירות
- הכותרת בתשובה מציינת מאיפה הגיע הפלט והכמה זמן לקח, למשל:
  `[LOCAL qwen2.5-coder:7b · 2.3s · score 0.22]` — מקומי, עלות אפס
  `[CLOUD claude-sonnet-4-6 · 4.1s · score 0.78]` — ענן, רק החשיבה עולה
  `[CHAIN ... ]` — הענן תכנן בקצרה והמקומי ייצר את הגוף
- `local-agent:stats` מציג סיכום: נתח-ענן, זמני תגובה, ועלות מוערכת

## הגדרות פרויקט

### פרויקט ללא .local-agent.json
אם בתיקיית העבודה **אין** `.local-agent.json` ועומד להיות שימוש ראשון באחד מכלי `local-agent` —
שאל את המשתמש שתי שאלות קצרות לפני שמפעילים את הכלי:
1. `priority`? (quality / cost / speed / balanced — ברירת מחדל = priority מהקונפיג הגלובלי)
2. מודל מקומי? (ברירת מחדל = local_model מהקונפיג הגלובלי)

לאחר מכן הרץ `python setup.py --project` כדי לשמור את הבחירה, **או** שאל את המשתמש אם לשמור עכשיו.
עשה זאת רק פעם אחת לסשן (אחרי שהמשתמש ענה — פעל עם הגדרות אלה ואל תשאל שוב).

### פרויקט עם .local-agent.json
בתחילת כל סשן שבו קיים `.local-agent.json` בתיקייה — הצג **שורת סיכום אחת** לפני כל פעולה:

```
פרויקט: <priority> · <local_model> · לשינוי: python setup.py --project
```

לדוגמה: `פרויקט: quality · qwen2.5-coder:14b · לשינוי: python setup.py --project`
