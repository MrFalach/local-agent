# local-agent

כלי שמפעיל את ה-AI שלך בחינם על המחשב שלך — וקורא לענן רק כשצריך.

כשאתה עובד עם Claude Code, כל בקשה שלך עוברת דרך local-agent. הוא מחליט אוטומטית: משימה פשוטה (כתיבת קוד, ריפקטור, תיעוד) — מועברת למודל שרץ אצלך בחינם. משימה מורכבת (ארכיטקטורה, אבטחה, באג עמוק) — עולה לענן רק כשיש צורך. אתה לא צריך לעשות כלום, הוא מסתדר לבד.

---

## מה בדיוק קורה עם כל בקשה?

```
אתה שואל את Claude Code משהו
              ↓
        local-agent מחליט
       ↙       ↓        ↘
   מקומי     chain       ענן
  (חינם)   (ענן מתכנן,  (רק כשצריך
           מקומי מבצע)   באמת)
```

- **מקומי** — כתיבת קוד, שינויים, בדיקות, תיעוד → חינם לחלוטין, מהיר
- **chain** — הענן כותב תכנית קצרה, המקומי מממש את הגוף → איכות גבוהה, עלות נמוכה
- **ענן** — שאלות מורכבות, ארכיטקטורה, אבטחה → רק כשאין ברירה

כל תשובה מגיעה עם שורת כותרת קצרה שאומרת מאיפה הגיעה:

```
[LOCAL qwen2.5-coder:14b · 1.8s · score 0.18]   ← מקומי, עלות אפס
[CLOUD claude-sonnet-4-6 · 3.4s · score 0.81]   ← ענן, רק החשיבה עולה
[CHAIN ... · 5.1s · score 0.74]                 ← שניהם יחד
```

## התקנה

**נדרש:** Claude Code, Python 3.10+.
**אופציונלי:** [Ollama](https://ollama.com) — בשביל המודל המקומי. בלעדיו הכל הולך לענן.

```bash
git clone https://github.com/MrFalach/local-agent.git
cd local-agent
pip install -r requirements.txt
python setup.py
```

האשף שואל אותך עד 4 שאלות:

1. גלובלי (כל הפרויקטים) או לפרויקט הנוכחי בלבד?
2. איזה מודל מקומי? (רשימה מותאמת אוטומטית למחשב שלך)
3. מה הכי חשוב — איכות / חיסכון / מהירות / מאוזן?
4. מפתח API של Anthropic (אם עוד אין)

בסוף הוא מגדיר את הכל ב-Claude Code. צריך להפעיל מחדש את Claude Code אחרי שגמרת.

## המודל מותאם למחשב שלך

האשף קורא כמה RAM יש לך ומציג רק מודלים שהמחשב שלך יכול להריץ בנוחות. מודלים גדולים מדי נעלמים אוטומטית מהרשימה.

| RAM | מודל מומלץ | הערות |
|-----|-----------|-------|
| פחות מ-8 GB | — | מצב ענן בלבד |
| 8–16 GB | qwen2.5-coder:7b | מודל קידוד מוצק |
| 16–32 GB | qwen2.5-coder:14b | איכות טובה יותר, עדיין מהיר |
| 32–64 GB | qwen2.5-coder:32b | איכות גבוהה מקומית |
| 64 GB ומעלה | qwen2.5-coder:72b | איכות קרובה לענן, בחינם |

> רק מודל אחד רץ בזיכרון בכל פעם. מעבר בין מודלים לוקח 20–40 שניות. עדיף לבחור אחד שמכסה את רוב הצרכים ולהשאיר את השאר לענן.

## עדיפות — ההגדרה האחת שקובעת הכל

| עדיפות | התנהגות | מתי לבחור |
|--------|---------|-----------|
| `quality` | עולה לענן בקלות | פרויקטים חשובים, קוד קריטי |
| `cost` *(ברירת מחדל)* | נשאר מקומי; ענן רק כשקשה | עבודה יומיומית, חיסכון |
| `speed` | הכי מהיר, ללא chain | עריכות מהירות, נפח גבוה |
| `balanced` | פיצול חכם | שימוש כללי |

## הגדרות לפרויקט ספציפי

רוצה הגדרות שונות לפרויקט מסוים? הרץ את זה בתוך תיקיית הפרויקט:

```bash
cd ~/הפרויקט-שלך
python /path/to/local-agent/setup.py --project
```

זה שואל 2 שאלות (עדיפות + מודל), יורש את שאר ההגדרות מהגלובל, ושומר קובץ `.local-agent.json` בתיקייה. Claude Code מזהה אותו אוטומטית בפתיחה הבאה.

---

## שכבות הגדרות

ניתן לדרוס כל הגדרה — הגבוה ברשימה גובר:

```
משתני סביבה                           (LOCAL_AGENT_PRIORITY, LOCAL_MODEL, ...)
  └── .local-agent.json               (תיקיית הפרויקט — נכתב על ידי setup.py --project)
        └── ~/.claude/local-agent/config.json   (גלובל — נכתב על ידי setup.py)
              └── ברירות מחדל מובנות
```

`ANTHROPIC_API_KEY` נשמר בקובץ `.env` בלבד, לא בקובצי JSON.

## קבצים בפרויקט

```
setup.py      אשף ההתקנה (זיהוי חומרה, קטלוג מודלים, מצב --project)
hardware.py   זיהוי RAM → המלצת מודל מתאים
config.py     טעינת קונפיג רב-שכבתי + פרופילי עדיפות
server.py     שרת MCP — נקודת הכניסה ל-Claude Code
router.py     לוגיקת ניתוב (ציון 0..1 לפי מילות מפתח ומורכבות)
providers.py  חיבור לאולמה + Anthropic + מסלול chain
telemetry.py  תיעוד JSONL + סיכום סטטיסטיקות
CLAUDE.md     הוראות ניתוב מוטמעות ל-Claude Code עצמו
```
